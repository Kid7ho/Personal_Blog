C, Python 3, Java 비교 분석과 각 언어의 가능성 탐구
Jin-Myoung Hyun · Research / ETC · 2024.01.21

다음은 고등학교 1학년 때, 교내 소논문 대회를 위해 컴퓨터공학과를 꿈꾸던 2명의 친구들과 함께 작성한 소논문이다. 논문에 대해 전혀 읽어보지 않았고, 글이라고는 보고서나 독후감만 썼던 당시에 어떤 생각과 어떤 방식을 통해 얼만큼의 신빙성과 참신함을 가진 실험을 설계했는지를 중점적으로 소논문을 감상해보도록 하자.

I. 서론

1. 연구 배경 및 목적

1) 연구 배경

우리는 각각 선호하고 사용하는 프로그래밍 언어가 다르다. 그래서 종종 가장 좋은, 효율적인, 또는 편리한 프로그래밍 언어가 무엇인가에 대해 논쟁을 벌이며 상대방이 선호하는 프로그래밍 언어를 비판하고 평가하곤 했다. 이러한 과정에서 각각의 프로그래밍 언어에 장단점이 있다는 것을 파악하게 되었고, 이를 보다 심도있게 연구하기 위해 2019년 현재 프로그래머들 사이에서 가장 많이 사용되고 있는 3가지 프로그래밍 언어의 특징들을 직접 실험을 통해 알아보고, 사용처를 선행 연구 조사를 통해 연구하기로 했다.

2) 연구 목적

2019년 기준 가장 많이 상용되고 있는 프로그래밍 언어 3가지를 비교 분석후, 각 언어가 어떤 분야에서 주로 사용되고, 또 왜 그러한 언어를 사용해야 하는가를 연구하여 결론을 도출하는 것을 목표로 한다.

2. 연구 방법 및 절차

1) 연구 대상: C, Java, Python3

2) 연구 방법 :

※ 본 연구는 C언어는 GCC.8.1.0, Java의 경우 JDK 10.0.1, Python 3은 3.6.5버전으로 시행되었다.

  1. 같은 결과를 도출하는 코드를 각 언어로 작성하여 그에 대한 실행 시간과 메모리 사용량을 검사 및 기록한다.
  2. 코드는 다음과 같이 구성된다 (반복문 실행 속도 비교를 위한 코드).
  3. 위 방법을 여러 번 반복하여 결과를 비교 분석한다.
  4. 비교 분석한 결과가 시사하는 바를 탐구하고, 결론을 도출한다.

3) 연구 문제 :

연구에 사용되는 3가지 프로그래밍 언어들을 이용해 명령을 수행할 때, 어떤 언어가 각각 어떤 명령을 수행할 때 가장 효율적이며, 그 이유는 무엇인가?

II. 이론적 배경 및 선행연구

우리는 이론적 배경 및 선행연구로 연구 대상이 되는 세 가지 프로그래밍 언어의 특징 등을 조사하였다.

1. C

1972년에 벨 연구소(Bell Labs)의 데니스 리치가 만든 프로그래밍 언어로, 운영체제 개발, 임베디드 시스템, 데이터 베이스 등 빠른 속도가 요구되는 분야에서 주로 쓰인다. 또한 C++, C# 등의 언어로 파생되어 게임 개발, 고성능 애플리케이션 개발 등의 분야에 주로 사용되고 있다. 예시로 C#은 Unity 게임 개발 엔진, Windows 운영 체제 개발에 사용되었으며, C++은 Unreal Engine, Mac 운영 체제 개발 등에 사용되었다.

2. Java

썬 마이크로시스템즈의 제임스 고슬링(James Gosling)과 다른 연구원들이 개발한 객체 지향적 프로그래밍 언어로, 대표적으로 모바일 어플을 만드는데 사용된다. 객체 지향 프로그래밍은 프로그램을 유연하고 변경이 용이하게 만들기 때문에 웹 서버 개발 같은 대규모 소프트웨어 개발 분야에 많이 사용된다.

3. Python

1991년 귀도 반 로섬(Guido van Rossum)이 발표한 고급 프로그래밍 언어로, 머신 러닝, 웹 프레임워크 개발, 계산과학 등에 사용된다. 다른 언어들에 비해 속도가 현저히 느리지만, 사업 분야 등 일반적인 컴퓨터 응용 환경에서는 속도가 그리 중요하지 않고, 빠른 속도를 요하는 프로그램의 경우에도 프로토타이핑한 뒤 빠른 속도가 필요한 부분만 골라서 C 언어 등으로 모듈화할 수 있다는 점,(ctypes, SWIG, SIP 등의 래퍼 생성 프로그램) 또 Pyrex, Psyco, NumPy 등을 이용하면 수치를 빠르게 연산할 수 있다는 점에서 현재 가장 많이 쓰이는 프로그래밍 언어로 자리잡았으며, 과학, 공학 분야에서도 많이 이용되고 있다.

III. 연구 과정 및 결과

1. 각 언어의 프로그램 실행 속도를 비교하기 위해 1부터 10000까지 값을 1씩 증가시키는 프로그램을 각 언어로 작성한 뒤, 실험 환경을 동일하게 유지하기 위해 jdoodle.com이라는 온라인 컴파일 사이트에서 같은 프로그램을 10번 실행해 보았다. 실험에 앞서, 우리는 선행 조사를 통해 습득한 자료를 바탕으로 C언어의 실행 속도가 가장 빠를 것이며, 그 다음이 Java, 마지막이 Python3가 될 것이라고 가설을 세웠다. 아래를 각 언어로 작성한 프로그램과 실험 결과를 표로 정리한 것이다.

실험 결과, C의 실행속도가 가장 빠르고, 그 뒤를 Python3, Java가 순서대로 이었다. 그러나 선행 조사를 통해 Python3는 인터프리터 언어로, 코드를 한 줄씩 읽어 내려가며 실행하는 프로그래밍 언어이며, 이로 인해 C나 Java보다 실행 속도가 빠를 수 없다는 것을 알게 되었다. 이러한 결과가 발생한 이유는 컴파일 시간이 포함되었기 때문으로, 반복문의 크기를 늘이면 파이썬의 실행 횟수가 늘어 예상대로의 결과가 나타날 것이라고 생각해 1부터 10000까지 값을 1씩 증가시키는 프로그램에서 1부터 10000000까지 값을 1씩 증가시키는 프로그램을 각 언어로 작성하여 10번씩 실행하여 실험 결과를 기록하였다.

IV. 결론

1. 연구 결과

1번째 실험에선 C, Java, Python3 순서로 실행 시간이 빠르다는 결과가 도출되었다. 이러한 결과로서 우리는 C가 고성능 프로그램 개발에 주로 사용되는 이유를 밝혀낼 수 있었고, C가 개발된 지 상당히 오래된 언어임에도 불구하고 아직도 보편적으로 쓰이고 있는 이유를 알 수 있었다. 2번째 실험에서는 Python3, C, Java 순서로 프로그래밍 언어가 효율적이고 생산성 있다는 결과가 도출되었다. 이 결과를 통해 앞으로는 압도적으로 코드의 양이 많은 대규모 프로젝트에서 효율성이 좋아 더 짧고 간단하게 코드를 작성할 수 있는 Python3가 현시점에선 대규모 프로젝트에서 가장 많이 사용되지만 효율성, 생산성 면에서 특출난 점이 없는 Java보다 더 많이 사용될 것이라고 예측했다. 우리의 연구는 IT 분야로의 진출을 꿈꾸는 청소년 또는 청년들이 어떤 프로그래밍 언어를 배워야 할지 갈피를 잡게 만들어줄 소박한 이정표가 되어줄 것이다.

2. 한계점

연구 과정의 1번 항목의 1번째 실험에서, 컴파일 시간을 배제하고 프로그램의 실행 시간만을 측정할 수 있는 수단을 마련하지 못했다.

연구 과정의 2번 항목의 연구에서 백준 사이트에서 제공하는 숏코딩 자료를 활용하였기 때문에 자료가 절대적임을 입증할 수 없다.

마무리

여러 컴퓨터 언어들의 성능을 소요 시간과 사용 메모리를 통해 분석한다는 점에서 주제 자체는 참신하지는 않지만, 교내 소논문의 주제로는 충분히 나쁘지 않다고 보인다. 하지만, 실험을 설계 및 진행하는 과정에서 'IV. 결론'에서도 언급한 것처럼 결과에 치명적일 수 있는 여러 문제점들이 존재한다. 다만, 이 부분은 고등학생의 지식과 자원으로는 크게 발전시키기에는 어렵기 때문에 실험을 설계하던 당시에도 고민을 하다가 적당히 현실과 타협한 것 같다.

전체적으로 고등학생이 작성한 것치고 퀄리티가 높은 작품이라고 판단되며, 어린 나이로 굉장히 좋은 경험을 했었다는 생각이 든다.
